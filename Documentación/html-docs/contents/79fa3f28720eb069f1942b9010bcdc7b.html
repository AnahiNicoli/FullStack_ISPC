Actores: Administrador, Google, Cliente y Usuarios externos.

Disparador: Necesidad de realizar la compra mediante pago en efectivo.

Descripción: Caso de Uso en donde el usuario realiza el pago en efectivo.

Precondición: El usuario decide realizar el pago mediante efectivo de sus artículos seleccionados.

Postcondición: Ninguna.

Flujo Normal

1 - El sistema verifica tipo de usuario. 2 - El usuario ejecuta el pago. 3 - El sistema confirma la compra y envía correo con comprobante de pago. 4 - El usuario puede ver detalles de la operación de compra en su carrito.

Flujo Alternativo

3. El usuario ejecuta mal el pago.

1-Sistema envía mensaje de realización de pago incorrecto. 2- Prosigue el punto 2 del flujo normal.

4 . El usuario no termina de realizar la compra.

El sistema envía un mensaje de correo electrónico de que la compra no fue realizada.
